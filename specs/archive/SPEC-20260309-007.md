## 1. Overview

vida gains two capabilities that require more screen real estate than the single-line palette provides:

1. **Chat view** — when an AI command runs, the palette transitions into a scrollable conversation view. No search bar; just a message history, a bottom input, and Escape/Ctrl+B to return to the palette. Multi-turn follow-up is supported.
2. **Notes** — a `:note` command opens a title + body form in the same chat-view layout. On Ctrl+S, vida writes a plain Markdown file to a user-configured directory. No vendor lock-in: works with Obsidian, Logseq, or any folder of `.md` files.

## 2. Functional Requirements

### FR-01 · Chat View Transition

| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | When an AI command result begins streaming, the UI transitions to chat view before the first token renders |
| FR-01b | Chat view hides the search entry and command row entirely |
| FR-01c | Chat view uses a `GtkStack` child page — the same `GtkApplicationWindow`, no new surface |
| FR-01d | The chat panel has the same max width (680px) and background as the palette panel |
| FR-01e | Transitioning back to palette view clears chat history (memory is per-session only) |

### FR-02 · Chat View Layout

| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Top section: scrollable message history (`GtkScrolledWindow` + `GtkBox`) |
| FR-02b | Messages alternate: user bubble (right-aligned, `.vida-msg-user`) and AI bubble (left-aligned, `.vida-msg-ai`) |
| FR-02c | AI message text wraps at panel width with word-char wrap; user message text wraps similarly |
| FR-02d | History scrolls to the latest message automatically as tokens stream in |
| FR-02e | Bottom section: single-line text entry (`.vida-chat-entry`) + send icon button |
| FR-02f | Panel height grows up to 560px; past that height, history scrolls internally |
| FR-02g | A small header bar shows the command name that started the session (e.g. `explain`) with an `×` button that maps to Escape |

### FR-03 · Navigation

| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | Escape in chat view transitions back to palette view and clears the search entry |
| FR-03b | Ctrl+B in chat view has identical behaviour to Escape (go back) |
| FR-03c | Ctrl+C in chat view copies the last AI response to clipboard |
| FR-03d | Enter in the chat entry sends the follow-up message |

### FR-04 · Multi-Turn Conversation

| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | The first turn uses the original command's system prompt (e.g. "translate") |
| FR-04b | Subsequent turns send the full message history plus the new user message |
| FR-04c | The system prompt for a command is preserved across turns for the session |
| FR-04d | While a response is streaming, the chat entry is disabled; re-enabled on completion |
| FR-04e | A new turn can be started before the previous stream completes (current stream is cancelled) |

### FR-05 · Notes Command

| Sub-ID | Requirement |
|--------|-------------|
| FR-05a | `:note` opens note form view (a specialisation of chat view layout) |
| FR-05b | Note form has: title entry at top (single line), body textarea below (auto-sizing, wrapping) |
| FR-05c | Optional tags field at bottom (comma-separated plain text) |
| FR-05d | Ctrl+S saves the note and closes the palette |
| FR-05e | Escape discards without saving |
| FR-05f | `:note` with text after it pre-fills the title field |
| FR-05g | While the notes directory is not configured, `:note` shows an inline error: "Set [notes] dir in config.toml" |

### FR-06 · Notes Storage

| Sub-ID | Requirement |
|--------|-------------|
| FR-06a | Notes are written as plain `.md` files; no proprietary format or API |
| FR-06b | Config key `[notes] dir` specifies the target directory; `~` is expanded |
| FR-06c | If `dir` is absent or empty, notes are disabled (FR-05g error) |
| FR-06d | `:note` with a title → `{dir}/{inbox_subdir}/{slug}.md` (default inbox_subdir: `"Inbox"`) |
| FR-06e | `:note` with no title → `{dir}/{daily_subdir}/{YYYY-MM-DD}.md` appended (default daily_subdir: `"Daily"`) |
| FR-06f | Title slug: lowercase, spaces→hyphens, non-alphanumeric stripped |
| FR-06g | File template configurable; default: `# {title}\n\ntags: {tags}\n\n{body}\n` |
| FR-06h | If the target file already exists and is a daily note, vida appends a `---` separator then the new content |
| FR-06i | Non-daily notes with a colliding filename get a numeric suffix: `slug-2.md`, `slug-3.md`, etc. |

## 4. Architecture

### 4.1 UI Structure

```
GtkApplicationWindow
└── GtkStack  (.vida-root-stack)
    ├── "palette"  ← existing palette panel (search + results)
    └── "chat"     ← new chat panel
        ├── header bar  (command name label + × button)
        ├── GtkScrolledWindow  (message history)
        │   └── GtkBox  (.vida-chat-history, VERTICAL)
        │       └── … message bubbles …
        └── GtkBox  (bottom input row)
            ├── GtkEntry  (.vida-chat-entry, hexpand)
            └── GtkButton  (send icon)
```

The note form reuses the chat panel layout with the message history area replaced by title + body + tags fields.

### 4.2 Message Bubbles

Each bubble is a `GtkBox` with a `GtkLabel` (wrap=TRUE, wrap_mode=WORD_CHAR). User bubbles align RIGHT; AI bubbles align LEFT. CSS classes:
- `.vida-msg-user` — slightly lighter background tint, right margin
- `.vida-msg-ai` — no tint, left margin

### 4.3 Multi-Turn IPC

Follow-up messages reuse the existing `query` IPC path with a new field `history []{"role": "user"|"assistant", "content": "..."}`. The daemon appends history to the messages array before calling the AI provider.

```json
{
  "type": "query",
  "input": "what about Python specifically?",
  "history": [
    {"role": "user",      "content": "explain windows"},
    {"role": "assistant", "content": "In computing, Windows refers to…"}
  ]
}
```

### 4.4 Notes Package

New `internal/notes` package:
- `Config` struct: `Dir`, `DailySubdir`, `InboxSubdir`, `Template`
- `Save(cfg Config, title, body, tags string) error` — resolves path, applies template, writes file
- `DailyPath(cfg Config) string` — returns path for today's daily note
- `SlugPath(cfg Config, title string) string` — returns path for titled note

---

## 5. Config Schema

```toml
[notes]
dir          = "~/Notes"
daily_subdir = "Daily"
inbox_subdir = "Inbox"
template     = "# {title}\n\ntags: {tags}\n\n{body}\n"
```

## 6. Test Scenarios

### SCN-01 — Chat view opens on AI command
**Given** the palette is open and `:explain linux` is entered
**When** the user presses Enter
**Then** the UI transitions to chat view with a user bubble "linux" and an AI bubble that streams the response

### SCN-02 — Escape returns to palette
**Given** chat view is open
**When** the user presses Escape
**Then** the UI transitions back to palette view with the search entry cleared

### SCN-03 — Ctrl+B returns to palette
**Given** chat view is open
**When** the user presses Ctrl+B
**Then** behaviour is identical to SCN-02

### SCN-04 — Follow-up question
**Given** chat view has one completed AI turn
**When** the user types a follow-up in the chat entry and presses Enter
**Then** a new user bubble appears and a new AI stream begins, history is preserved

### SCN-05 — Chat entry disabled during streaming
**Given** an AI stream is in progress
**Then** the chat entry is non-interactive; it becomes active again when streaming ends

### SCN-06 — Ctrl+C copies last AI response
**Given** chat view has at least one AI message
**When** the user presses Ctrl+C
**Then** the last AI message text is copied to clipboard

### SCN-07 — :note opens note form
**Given** the palette is open
**When** the user enters `:note` and presses Enter
**Then** the note form opens with focus on the title entry

### SCN-08 — :note with pre-filled title
**Given** the palette is open
**When** the user enters `:note my idea` and presses Enter
**Then** the note form opens with "my idea" pre-filled in the title

### SCN-09 — Ctrl+S saves titled note
**Given** the note form has title "my idea" and body "some text"
**When** the user presses Ctrl+S
**Then** `{dir}/Inbox/my-idea.md` is created with the default template applied and palette closes

### SCN-10 — Ctrl+S saves daily note
**Given** the note form has no title and body "quick thought"
**When** the user presses Ctrl+S
**Then** `{dir}/Daily/YYYY-MM-DD.md` is created (or appended if it exists) and palette closes

### SCN-11 — Escape discards note
**Given** the note form has unsaved content
**When** the user presses Escape
**Then** no file is written and the palette closes

### SCN-12 — No dir configured shows error
**Given** `[notes] dir` is not set in config
**When** the user enters `:note` and presses Enter
**Then** an inline error "Set [notes] dir in config.toml" is shown; no form opens

### SCN-13 — Daily note appends on collision
**Given** `{dir}/Daily/YYYY-MM-DD.md` already exists
**When** a new daily note is saved
**Then** `---\n{new content}` is appended to the existing file

### SCN-14 — Titled note gets numeric suffix on collision
**Given** `{dir}/Inbox/my-idea.md` already exists
**When** a note with title "my idea" is saved
**Then** the file is saved as `{dir}/Inbox/my-idea-2.md`
