## 1. Overview

vida gains a fuzzy file search mode triggered by a `~` prefix. Typing `~readme` finds files matching "readme" across configured directories. Results display filename + truncated path; Enter opens the file via `xdg-open`. No network, no AI, no daemon round-trip for indexing — the file index is built at daemon startup and refreshed on reload.

---

## 2. Functional Requirements

### FR-01 · Trigger & Detection

| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | File search activates when the query starts with `~` followed by at least one non-whitespace character |
| FR-01b | The `~` prefix is stripped before matching; `~readme` searches for "readme" |
| FR-01c | File search runs after command mode (`:`) and unit conversion/calc/shortcuts, before app search |
| FR-01d | If the `~`-prefixed query yields no results, the UI shows an empty state (does NOT fall through to app search) |
| FR-01e | A bare `~` with no further input (or only spaces) returns no results |

### FR-02 · Index & Scope

| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Default search root: `$HOME` (recursive) |
| FR-02b | Config key `[files] dirs` overrides the search roots with an explicit list of absolute paths |
| FR-02c | Hidden files and directories (names starting with `.`) are skipped by default |
| FR-02d | Config key `[files] include_hidden = true` enables indexing of hidden files |
| FR-02e | The following directory names are always excluded regardless of config: `node_modules`, `.git`, `.svn` |
| FR-02f | Files larger than 500 MB are excluded from the index |
| FR-02g | The index is built asynchronously at daemon startup; file search returns empty results until indexing is complete |
| FR-02h | `vida reload` triggers a full index rebuild |
| FR-02i | Symlinks are followed one level deep; cycles are not followed |

### FR-03 · Matching & Ranking

| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | Matching uses case-insensitive fuzzy scoring on the file **basename** only |
| FR-03b | Results are sorted by match score descending, then alphabetically by basename |
| FR-03c | A maximum of 10 results are returned |
| FR-03d | Exact prefix matches score highest, followed by substring matches, then fuzzy subsequence matches |

### FR-04 · Result Display

| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | Each result row shows: file icon (mime-type based), basename (bold), truncated parent path (muted) |
| FR-04b | The path is truncated to 40 characters from the right, prefixed with `…` if truncated |
| FR-04c | Home directory prefix in paths is replaced with `~` (e.g. `/home/dina/docs` → `~/docs`) |
| FR-04d | Row icon uses the same GTK icon lookup as app results; fallback icon: `text-x-generic` |
| FR-04e | Selected row highlights with the same accent style as app results |

### FR-05 · Actions

| Sub-ID | Requirement |
|--------|-------------|
| FR-05a | Enter on a selected file executes `xdg-open <path>` and closes the palette |
| FR-05b | If `xdg-open` is unavailable, execute `$BROWSER <path>` as fallback; if both are unavailable, do nothing |
| FR-05c | Ctrl+C on a selected file copies the absolute path to clipboard |
| FR-05d | Arrow keys navigate the list; keyboard behaviour is identical to app results |

## 3. Technical Requirements

### TR-01 · Package `internal/files`

```go
// File represents a single indexed file.
type File struct {
    Path string  // absolute path
    Name string  // basename
    Icon string  // GTK icon name (mime-type derived, may be empty → fallback)
}

// Index holds the in-memory file index.
type Index struct { ... }

// BuildIndex walks dirs and returns a populated Index.
// Hidden files/dirs are skipped unless includeHidden is true.
// Always skips: node_modules, .git, .svn.
func BuildIndex(dirs []string, includeHidden bool) (*Index, error)

// Search returns up to max Files whose basenames fuzzy-match query (case-insensitive).
// Results are sorted by score descending.
func (idx *Index) Search(query string, max int) []File

// Len returns the number of indexed files.
func (idx *Index) Len() int
```

### TR-02 · Router Changes (`internal/router/router.go`)

New kind and result field:

```go
KindFileList Kind = "file_list"

type FileEntry struct {
    Path string
    Name string
    Icon string
}

// Result gains:
Files []FileEntry
```

Priority chain insertion — between shortcuts (3) and apps (4):

```go
// 3.5. File search — "~" prefix
if strings.HasPrefix(input, "~") {
    query := strings.TrimPrefix(input, "~")
    query = strings.TrimSpace(query)
    if query != "" && r.fileIndex != nil {
        matched := r.fileIndex.Search(query, 10)
        entries := make([]FileEntry, len(matched))
        for i, f := range matched {
            entries[i] = FileEntry{Path: f.Path, Name: f.Name, Icon: f.Icon}
        }
        return Result{Kind: KindFileList, Files: entries}
    }
    return Result{Kind: KindEmpty}
}
```

New Router option:

```go
func WithFileIndex(idx *files.Index) Option
```

### TR-04 · Daemon (`cmd/vida-daemon/main.go`)

- Build file index asynchronously after app index (same pattern as `indexApps()`)
- Pass `WithFileIndex(fileIdx)` to router on reload
- Handle `KindFileList` in `handleQuery`:

```go
case router.KindFileList:
    names := make([]string, len(result.Files))
    paths := make([]string, len(result.Files))
    icons := make([]string, len(result.Files))
    for i, f := range result.Files {
        names[i] = f.Name
        paths[i] = f.Path
        icons[i] = f.Icon
    }
    resp.Kind = "file_list"
    resp.Message = strings.Join(names, "\n")
    resp.Paths  = strings.Join(paths, "\n")
    resp.Icons  = strings.Join(icons, "\n")
```

### TR-05 · IPC (`internal/ipc/ipc.go`)

Add `Paths string` field to `Message` struct (newline-delimited absolute file paths).

### TR-06 · UI (`cmd/vida-ui/main.go`)

- Handle `"file_list"` result kind in the switch
- Render using same GTK `ListBox` pattern as `"app_list"`: icon + name row
- Show truncated parent path as subtitle (muted label below name)
- Enter action: `exec.Command("xdg-open", path).Start()`, close palette
- Ctrl+C action: write `path` to clipboard
- Store `currentFilePaths []string` parallel to row indices (same pattern as `currentAppExecs`)

## 4. Test Scenarios

### SCN-01 · Trigger detection
- `~readme` → triggers file search, query = "readme"
- `~ readme` → triggers file search, query = "readme" (trimmed)
- `~` alone → returns empty
- `readme` (no prefix) → does NOT trigger file search (falls through to app search)
- `:files` → command mode, not file search

### SCN-02 · Hidden file exclusion
- Index built with `includeHidden=false`: `.bashrc` not in results for `~bash`
- Index built with `includeHidden=true`: `.bashrc` appears in results for `~bash`
- `node_modules/index.js` never indexed regardless of `includeHidden`

### SCN-03 · Fuzzy matching & ranking
- Files: `README.md`, `readme_old.txt`, `my_readme.md`
- Query "readme" → `README.md` scores highest (exact prefix, case-insensitive)

### SCN-04 · Result count cap
- Index with 50 matching files → only 10 returned

### SCN-05 · Router priority — file search before apps
- Input `~fire` with apps containing "Firefox" and a file named `firewall.conf`
- Result kind = `file_list`, not `app_list`

### SCN-06 · No results
- Input `~zzznomatch` → `KindEmpty` result (not `KindFileList` with empty slice)

### SCN-07 · Custom dirs config
- `[files] dirs = ["/tmp/testdir"]` → only files under `/tmp/testdir` are indexed

### SCN-08 · Index.Len
- After `BuildIndex` over a directory with 5 files → `Len() == 5`

## 5. Out of Scope

- Real-time filesystem watching (inotify) — reload on `vida reload` is sufficient for v1
- Directory navigation (cd-style browsing)
- File preview
- File operations (rename, delete, move)
- Search inside file content (full-text search)
- Searching by metadata (size, date, type filter)
