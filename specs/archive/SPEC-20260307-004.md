## Functional Requirements

### FR-01: App Selection
- **FR-01a:** Arrow Down / Arrow Up move selection through result rows
- **FR-01b:** Selected row has a distinct highlight (brighter than hover)
- **FR-01c:** Selection wraps: Down from last → first, Up from first → last
- **FR-01d:** Selection resets to row 0 when query text changes
- **FR-01e:** Clicking a row selects and immediately launches it

### FR-02: Launching
- **FR-02a:** Enter key launches the currently selected row (or row 0 if none selected)
- **FR-02b:** App is launched via `g_app_info_launch` using the `.desktop` file's GAppInfo
- **FR-02c:** Launch is fire-and-forget — launcher does not wait for the app
- **FR-02d:** Launcher hides immediately after a successful launch
- **FR-02e:** Entry is cleared on hide (existing behaviour preserved)

### FR-03: Exec Handling
- **FR-03a:** `%u`, `%U`, `%f`, `%F`, `%i`, `%c`, `%k` field codes are stripped from the Exec string before launch
- **FR-03b:** Apps with `Terminal=true` in their `.desktop` file are launched in the user's default terminal

### FR-04: Non-app Rows
- **FR-04a:** Enter on a calc result copies the value to clipboard
- **FR-04b:** Enter on a URL/shortcut result opens the URL (same as the Open button)
- **FR-04c:** Enter on an AI result copies the text to clipboard

## Technical Requirements

### TR-01: Selected Index State
- **TR-01a:** `selectedIdx int` (default -1 = none) stored as a Go package-level var, only touched via `gtkIdle`
- **TR-01b:** Reset to -1 on `vida_results_clear`
- **TR-01c:** `goOnKeyPressed` handles `GDK_KEY_Down` and `GDK_KEY_Up`, calls `C.vida_select_row`

### TR-02: Row Highlight CSS
- **TR-02a:** Selected row gets CSS class `vida-row-selected`
- **TR-02b:** `.vida-row-selected` background: `rgba(255, 255, 255, 0.13)`

### TR-03: Launch via GAppInfo
- **TR-03a:** `apps.Index` stores the `.desktop` file ID (e.g. `firefox.desktop`)
- **TR-03b:** IPC response for `app_list` includes the desktop ID alongside the name (new `ids` field, `\n`-separated, parallel to `message`)
- **TR-03c:** `vida_launch_app(desktop_id)` calls `g_app_info_launch` with the GDesktopAppInfo
- **TR-03d:** On launch failure, launcher stays visible (does not hide)

### TR-04: Result Kind Tracking
- **TR-04a:** Current result kind (`calc`, `shortcut`, `app_list`, `ai_stream`) stored as Go package-level var
- **TR-04b:** Enter key behaviour branches on this kind

## Scenarios

### SCN-01: Arrow key navigation
**Given** app results are showing
**When** user presses Down
**Then** first row gets selected highlight
**When** user presses Down again
**Then** second row gets selected highlight

### SCN-02: Enter launches app
**Given** "Firefox" is selected
**When** user presses Enter
**Then** Firefox launches as a detached process
**And** the launcher hides

### SCN-03: Click launches app
**Given** app results are showing
**When** user clicks a row
**Then** that app launches and launcher hides

### SCN-04: Enter on calc copies value
**Given** calc result "4" is showing
**When** user presses Enter
**Then** "4" is copied to clipboard

### SCN-05: Enter on URL opens it
**Given** shortcut result is showing
**When** user presses Enter
**Then** URL opens in browser (same as clicking Open)

### SCN-06: Selection resets on new query
**Given** row 2 is selected
**When** user types another character
**Then** selection resets to none

## Architecture Notes

- `apps.App` gains an `ID string` field (desktop file ID, e.g. `firefox.desktop`)
- Daemon IPC response for `app_list` gains a parallel `\n`-separated `ids` field
- UI stores `[]string` of current app IDs alongside result rows
- `vida_launch_app` is a new C function in `ui.c`
- `vida_select_row(box, idx)` adds/removes `vida-row-selected` CSS class
- `goOnKeyPressed` handles Down/Up (currently only handles Escape/Enter)
