## 1. Overview

vida gains a command mode triggered by typing `:` as the first character in the search field. Commands are discrete named actions — system operations, user-defined shell snippets, or AI prompts with fixed system prompts. They appear in their own result list, never mixed with apps. Output either silently closes the palette or streams into the result area. This makes vida a genuine command palette, not just an app launcher.

---

## 2. Functional Requirements

### FR-01 · Mode Switching
| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | Input starting with `:` (and only `:` as the first character) activates command mode |
| FR-01b | All other input uses existing calc → shortcut → app → AI routing unchanged |
| FR-01c | Deleting the `:` returns to normal mode |
| FR-01d | Escape in command mode clears input and closes palette (same as normal mode) |
| FR-01e | Placeholder text in command mode: `"Type a command…"` |

### FR-02 · Command List
| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Typing `:` alone shows all available commands |
| FR-02b | Typing `:<query>` fuzzy-filters commands by name and description |
| FR-02c | Command rows display: icon (32px) + name (bold) + description (muted) |
| FR-02d | Arrow keys navigate the command list; Enter runs the selected command |
| FR-02e | Commands are ordered: built-in system → built-in AI → user-defined |
| FR-02f | If no commands match the query, show a single muted row: `"No commands match"` |

### FR-03 · Built-in System Commands
| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | The following built-ins are always available: `lock`, `sleep`, `reboot`, `shutdown`, `reload-hypr`, `kill-window`, `reload-vida` |
| FR-03b | Each built-in has a default exec string; table in TR-02 |
| FR-03c | Built-in exec strings are overridable via `[commands.builtins.<name>] exec = "..."` in config |
| FR-03d | `reload-vida` sends the `reload` IPC message instead of running a shell command |
| FR-03e | System commands run asynchronously; palette closes immediately on Enter (output = none) |

### FR-04 · User-Defined Commands
| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | User commands defined as `[[commands.user]]` blocks in config.toml |
| FR-04b | Required fields: `name`, `exec`; optional: `desc`, `icon`, `output` |
| FR-04c | `output` defaults to `"none"` if omitted |
| FR-04d | `{input}` in `exec` is replaced with text typed after the command name (URL-encoded for safety is NOT applied — raw substitution, user's responsibility) |
| FR-04e | User commands are reloaded when daemon receives `reload` message |
| FR-04f | Duplicate names: user commands override built-ins of the same name |

### FR-05 · Output Modes
| Sub-ID | Requirement |
|--------|-------------|
| FR-05a | `output = "none"`: exec runs in background, palette closes on Enter, no result shown |
| FR-05b | `output = "palette"`: exec stdout is captured and displayed in the result area as plain text |
| FR-05c | For `output = "palette"`, output is truncated at 100 lines; a muted note `"(truncated)"` is appended if exceeded |
| FR-05d | Exec stderr is discarded for `output = "palette"` (only stdout is shown) |
| FR-05e | If exec exits non-zero with `output = "palette"`, stderr is shown instead with an error style |

### FR-06 · AI Commands
| Sub-ID | Requirement |
|--------|-------------|
| FR-06a | Built-in AI commands: `translate`, `explain`, `define`, `fix`, `summarize` |
| FR-06b | AI commands use the existing AI streaming infrastructure (same `ai_stream` IPC flow) |
| FR-06c | Text typed after the command name (`:translate hello world` → `hello world`) is the AI query input |
| FR-06d | Each AI command injects a fixed system prompt prefix before the user's input (see TR-03) |
| FR-06e | AI command results stream into the result area identically to a normal AI query |
| FR-06f | AI commands require at least 1 character of input after the command name; typing `:translate` alone shows the command description, does not invoke AI |

### FR-07 · Keyboard Actions on Results
| Sub-ID | Requirement |
|--------|-------------|
| FR-07a | `Ctrl+C` on any focused result row copies its text content to the clipboard |
| FR-07b | A brief HUD-style status label `"Copied"` appears for 1.5 seconds after copy |
| FR-07c | `Ctrl+C` works for command output, AI command output, and existing calc/AI results |

## 3. Technical Requirements

### TR-01 · Router Changes (`internal/router/router.go`)

Add `KindCommandList` result kind. The router's first check (before calc) must detect `:` prefix:

```go
// Priority 0 — command mode (before calc)
if strings.HasPrefix(input, ":") {
    query := strings.TrimPrefix(input, ":")
    return Result{Kind: KindCommandList, CommandQuery: query}
}
```

New fields on `Result`:
```go
KindCommandList Kind = "command_list"

// In Result struct:
CommandQuery    string        // text after `:`, used for filtering
Commands        []CommandEntry
```

```go
type CommandEntry struct {
    Name   string
    Desc   string
    Icon   string
    Kind   string // "system" | "ai" | "user"
    Exec   string // empty for ai commands
    Output string // "none" | "palette"
}
```

### TR-02 · Command Registry (`internal/commands/commands.go`)

New package. Responsibilities:
- Hold the list of built-in + user commands
- Fuzzy-filter by name+desc
- Execute system/user commands (spawn subprocess, capture stdout)

**Built-in table:**
| Name | Exec | Output |
|------|------|--------|
| `lock` | `loginctl lock-session` | none |
| `sleep` | `systemctl suspend` | none |
| `reboot` | `systemctl reboot` | none |
| `shutdown` | `systemctl poweroff` | none |
| `reload-hypr` | `hyprctl reload` | none |
| `kill-window` | `hyprctl kill` | none |
| `reload-vida` | *(IPC reload, handled in daemon)* | none |

**Execution:**
```go
func (r *Registry) Run(name, input string) (output string, err error)
```
- Substitutes `{input}` in exec string
- For `output=none`: `exec.Command("sh", "-c", exec).Start()` (detached)
- For `output=palette`: `exec.Command("sh", "-c", exec).Output()` (captured, timeout 10s)
- `reload-vida` returns a sentinel that daemon handles directly

### TR-03 · AI Command System Prompts

Each AI command prepends a system prompt to the user's input before sending to the provider:

| Command | System prompt |
|---------|--------------|
| `translate` | `"Translate the following text to English. If a target language is specified as the first word (e.g. 'es', 'fr', 'de'), translate to that language instead. Output only the translation, no commentary."` |
| `explain` | `"Explain the following in 2–3 plain sentences suitable for a command palette display. No markdown headers. Be concise."` |
| `define` | `"Give a dictionary definition of the following word or phrase. Include part of speech, primary definition, and brief etymology if known. No markdown headers."` |
| `fix` | `"Fix the spelling and grammar of the following text. Output only the corrected text, nothing else."` |
| `summarize` | `"Summarize the following text as concise bullet points. No markdown headers. Max 5 bullets."` |

### TR-04 · IPC Protocol Additions

New `command_list` result kind sent by daemon for `:` queries:
```json
{
  "type": "result",
  "kind": "command_list",
  "id": "<uuid>",
  "commands": "[{\"name\":\"lock\",\"desc\":\"Lock screen\",\"icon\":\"system-lock-screen\",\"kind\":\"system\"},...]\n"
}
```
Commands serialized as JSON array in the `message` field (newline-terminated, consistent with existing pattern).

New message from UI → daemon for execution:
```json
{ "type": "run_command", "id": "<uuid>", "name": "dict", "input": "ephemeral" }
```

Daemon responses:
```json
{ "type": "command_done", "id": "<uuid>" }                    // output=none
{ "type": "command_result", "id": "<uuid>", "value": "..." }  // output=palette
{ "type": "command_error", "id": "<uuid>", "message": "..." } // exec failed
```
AI commands reuse existing `query` → `ai_stream` → `token` / `done` flow with system prompt injection.

### TR-06 · Config Schema (`internal/config/config.go`)

```go
type CommandsConfig struct {
    Builtins map[string]BuiltinOverride `toml:"builtins"`
    User     []UserCommand              `toml:"user"`
}

type BuiltinOverride struct {
    Exec string `toml:"exec"`
}

type UserCommand struct {
    Name   string `toml:"name"`
    Desc   string `toml:"desc"`
    Icon   string `toml:"icon"`
    Exec   string `toml:"exec"`
    Output string `toml:"output"` // "none" | "palette", default "none"
}
```
