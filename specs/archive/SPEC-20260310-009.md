# SPEC-20260310-009: Answer Bar for Calc & Convert Results

## 1. Overview

Calc and unit conversion results currently render as app-style rows — a flat button with hover state that looks identical to a search result. This is wrong: a computed answer is not a result to navigate to, it's an immediate response to what was typed.

A dedicated **answer bar** sits between the search entry and the results list. It has its own visual identity — monospace value, muted type label, no interactive styling — making it unmistakably distinct from app rows.

---

## 2. Functional Requirements

### FR-01 · Answer Bar Display

| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | A dedicated answer bar is placed between the search entry and the separator/results list |
| FR-01b | The answer bar is hidden by default and shown only for calc and convert results |
| FR-01c | The answer bar shows the result value in a large monospace font |
| FR-01d | A small muted label below the value identifies the type: `CALC` or `CONVERT` |
| FR-01e | The answer bar has no hover state, no button, and is not keyboard-navigable |
| FR-01f | When the answer bar is visible, the separator and results list are hidden |

### FR-02 · Interaction

| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Enter copies the result value to clipboard (same behaviour as before) |
| FR-02b | Ctrl+C copies the result value to clipboard |
| FR-02c | Clearing the search entry hides the answer bar |

## 3. Technical Requirements

### TR-01 · C Functions

```c
// vida_build_window gains a third output: out_answer
GtkWidget *vida_build_window(GtkApplication *app,
                              GtkWidget **out_entry,
                              GtkWidget **out_results,
                              GtkWidget **out_answer);

// Show a value in the answer bar with a type label ("CALC" or "CONVERT")
void vida_answer_set(GtkWidget *answer, const char *value, const char *type);

// Hide the answer bar
void vida_answer_clear(GtkWidget *answer);
```

### TR-02 · Widget Structure

```
panel (GtkBox vertical)
├── entry_row          ← unchanged
├── answer_bar         ← NEW (.vida-answer, hidden by default)
│   ├── value_label    ← .vida-answer-value
│   └── type_label     ← .vida-answer-type
├── separator          ← hidden when answer bar is visible
└── results            ← hidden when answer bar is visible
```

### TR-03 · CSS

```css
.vida-answer {
  padding: 14px 20px 12px 20px;
  border-top: 1px solid rgba(255,255,255,0.06);
  border-bottom: 1px solid rgba(255,255,255,0.06);
}
.vida-answer-value {
  font-family: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', monospace;
  font-size: 24px;
  font-weight: 600;
  color: #ffffff;
}
.vida-answer-type {
  font-family: 'Inter', system-ui, sans-serif;
  font-size: 11px;
  color: rgba(255,255,255,0.28);
  text-transform: uppercase;
  letter-spacing: 1px;
  margin-top: 2px;
}
```

### TR-04 · main.go

- Add `gAnswer *C.GtkWidget` alongside `gEntry` and `gResults`
- `calc` and `convert` response cases call `vida_answer_set(gAnswer, val, "CALC"/"CONVERT")` and `vida_results_clear(gResults)` + hide separator
- All other result kinds call `vida_answer_clear(gAnswer)` before showing results
- `vida_results_clear` already hides the separator; `vida_answer_clear` must also ensure separator visibility is managed correctly
