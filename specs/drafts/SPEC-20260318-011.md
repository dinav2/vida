## Problem Statement

Linux/Wayland has no built-in clipboard history. Once you copy something new, the previous
entry is gone. Users who rely on keyboard-driven workflows need quick access to recent
clipboard entries without reaching for the mouse or switching apps.

## Solution

vida daemon continuously monitors the Wayland clipboard and stores entries in SQLite.
Typing `:cb` or `:clipboard` in the command palette dismisses the palette and opens a
dedicated clipboard history window — a taller, scrollable panel showing all stored entries.

The clipboard window is a second window type inside `vida-ui`, shown/hidden independently
via IPC. It is distinct from the main palette so users can browse and scan history
comfortably.

### Clipboard Window Layout

```
┌─────────────────────────────────────┐
│  🔍 Search clipboard...             │  ← search bar (auto-focused)
├─────────────────────────────────────┤
│  just copied this thing             │  ← newest first
│  some other copied text             │
│  https://example.com/long/url       │
│  another entry here                 │
│  ...                                │
└─────────────────────────────────────┘
```

- ~400px wide, ~500px tall (shows ~10–15 entries)
- Positioned same as the main palette (centered, near top)
- Escape closes it
- Enter on selected entry: writes to clipboard, closes window
- Search bar filters entries live as you type

### Entry Points

| Trigger | Behaviour |
|---------|-----------|
| `:cb` in palette | palette closes, clipboard window opens |
| `:clipboard` in palette | same |
| `vida clipboard` CLI | opens clipboard window directly |

## Notes Storage

Clipboard entries are stored in a new `clipboard_history` SQLite table in the vida database:

```sql
CREATE TABLE clipboard_history (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    content   TEXT    NOT NULL,
    copied_at INTEGER NOT NULL,  -- unix timestamp
    pinned    INTEGER NOT NULL DEFAULT 0
);
```

## Configuration

```toml
[clipboard]
enabled      = true
max_entries  = 500
max_age_days = 30
ignore       = []   # list of regex patterns to suppress (e.g. password managers)
```

## Key Technical Notes

- Clipboard monitoring: `wl-paste --watch` subprocess launched by daemon at startup
- Paste: write selected entry back via `wl-copy`, window closes — user's app already has it
- Duplicate detection: if new entry equals the most recent entry, do not store again
- Large entries (>500 chars): truncated in list display, full content restored on paste
- Image entries: store `[image]` placeholder text, skip actual image data for now
- Ignore patterns: regex match against content before storing; matching entries are dropped silently
- Pruning: on each new insert, delete entries beyond `max_entries` and older than `max_age_days`
  (pinned entries are exempt from both limits)
