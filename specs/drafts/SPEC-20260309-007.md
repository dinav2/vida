## Problem Statement

Some commands need more than a single-line input. Writing a note, translating a paragraph,
or fixing a long block of text cannot be done comfortably in a 640px single-line entry.
The palette needs a textarea that grows with the content — not a fixed expanded size, but
a panel that breathes naturally as you type.

## Solution

When a command that requires multi-line input is triggered, the single-line entry is replaced
by a `GtkTextView` textarea. The panel height auto-sizes to fit the content:
- Starts at the entry's normal height (~56px)
- Grows smoothly as lines are added (GTK natural height)
- Caps at ~500px, then scrolls internally past that
- Shrinks back as lines are deleted

No mode jump, no fixed "expanded" state — it just grows.

**AI split view** (`:translate`, `:fix`, `:summarize` with large input):
- Left column: auto-sizing textarea (input)
- Right column: AI output streams in real time as text settles (debounced)
- Ctrl+C copies the output

**Note form view** (`:note`):
- Title field (single line, at top)
- Body: auto-sizing textarea below
- Tags field (optional, comma-separated, at bottom)
- Ctrl+S saves, Escape discards

## Notes Storage

Notes are plain markdown files written to a configurable directory. No app-specific API —
vida just writes `.md` files. Works with Obsidian, Logseq, or any directory of markdown:

```toml
[notes]
dir          = "~/Notes"      # required — point at your vault/folder
daily_subdir = "Daily"        # :note with no title → Daily/YYYY-MM-DD.md
inbox_subdir = "Inbox"        # :note with title    → Inbox/slugified-title.md
template     = "# {title}\n\ntags: {tags}\n\n{body}"
```

## Key Technical Notes

- `GtkTextView` reports natural height via `gtk_widget_measure`; hook into
  `GtkTextBuffer::changed` to re-measure and call `gtk_widget_set_size_request` on panel
- Cap: `max_height = 500` (configurable); past cap, wrap in `GtkScrolledWindow`
- Split pane for AI commands: `GtkPaned` with left textarea + right label/AI output column
- File naming: slugified title if provided, else `YYYY-MM-DD-HH-MM.md`
- Obsidian daily note convention: `daily_subdir/YYYY-MM-DD.md` — appends if file exists
