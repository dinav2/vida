# PRD: vida UI Styling — Mac-Style Launcher Aesthetic

**ID:** SPEC-20260307-003
**Stage:** DRAFT
**Date:** 2026-03-07
**Priority:** P1

---

## Overview

vida should feel like a premium personal assistant, not a utility tool.
The visual design should evoke the polish of Alfred or macOS Spotlight —
frosted glass, generous whitespace, clean typography, and a centered floating panel.

Currently the window is anchored top/left/right with no styling — raw GTK defaults.
This spec replaces that with a styled, centered launcher.

---

## Goals

- Look immediately "premium" on first open
- Feel light and unobtrusive — it appears, you use it, it disappears
- No visual clutter; results appear only when needed
- Works on dark Wayland compositors (Hyprland)

---

## Design Direction

### Window
- Centered horizontally and vertically (or slightly above center)
- Fixed width: ~640px
- Height: auto (grows with results, max ~400px)
- Rounded corners: 16px
- Dark frosted/semi-transparent background: `rgba(20, 20, 25, 0.92)`
- Subtle border: 1px `rgba(255,255,255,0.08)`
- Drop shadow via compositor (no GTK shadow needed)

### Search Entry
- Large, prominent input — font size 20px
- No visible border/box — blends into panel background
- Padding: 20px horizontal, 16px vertical
- Placeholder text: muted color `rgba(255,255,255,0.35)`
- Search icon (🔍 or symbolic `system-search`) left of text, muted
- Cursor: white

### Results
- Appear below entry with a subtle 1px separator line
- Each result row: 44px tall, 20px horizontal padding
- Hover/selected state: `rgba(255,255,255,0.07)` background, 8px border-radius
- Text: white, 15px, regular weight
- Secondary text (type label e.g. "Calculator", "App"): muted, 12px, right-aligned
- Max 6 rows visible before scrolling

### Typography
- Font: system-ui or "Inter" if available, fallback to sans-serif
- No bold anywhere except maybe the matched portion

### Animation
- Window appears instantly (layer-shell constraint — no fade possible without compositor rule)

---

## Non-Goals

- Light mode (dark only for now)
- Custom icon rendering per app
- Animations/transitions (Wayland layer-shell limitation)
- Theming system / config for colors

---

## Acceptance Criteria

- AC-S1: Window is horizontally centered, vertically at ~35% from top
- AC-S2: Window has rounded corners visible against desktop
- AC-S3: Background is dark semi-transparent (not opaque white/grey GTK default)
- AC-S4: Search entry text is large (≥18px) and readable
- AC-S5: Results rows have visible hover highlight
- AC-S6: No GTK title bar or window decorations visible
- AC-S7: Calc/shortcut/app results show a muted type label on the right
