# PRD: SPEC-20260308-005 — App Icons in Launcher Results

## Problem Statement

App result rows in vida's launcher display only the application name as plain text. Users cannot visually identify apps at a glance, which slows navigation in a fast command palette. Every major launcher (Alfred, Raycast, GNOME Activities) shows app icons for this reason.

## Solution

Load the application icon from each app's `.desktop` entry using `GDesktopAppInfo` / `GIcon` and render it as a 48×48 px image to the left of the app name in each launcher result row. When no icon is available, fall back to a generic `application-x-executable-symbolic` icon.

## Scope

**In scope:**
- 48×48 px icon rendered left of app name in each app result row
- Icon loaded via `GDesktopAppInfo` → `g_app_info_get_icon()` → `GtkImage`
- Fallback to `application-x-executable-symbolic` if icon is NULL or fails to load
- Icon is rounded (CSS `border-radius`) to match macOS launcher aesthetic
- Row height expands to accommodate icon (min 56px)
- No changes to daemon, IPC, or Go code — UI-only change in `ui.c`

**Out of scope:**
- Custom icon overrides in config
- Icon caching / pre-loading at startup
- Animated icons or badge overlays

## Requirements

| ID | Requirement |
|----|-------------|
| R-01 | Each app row displays a 48×48 px icon left of the app name |
| R-02 | Icon is loaded from the app's `.desktop` Icon field via `GDesktopAppInfo` |
| R-03 | If icon resolution fails, render `application-x-executable-symbolic` at 48px |
| R-04 | Icons are rendered with 6px border-radius (rounded square) |
| R-05 | Row min-height is 56px (up from 44px) |
| R-06 | Icon has 12px left margin, 8px right gap to app name |
| R-07 | App name label remains left-aligned, ellipsized at end |
| R-08 | "App" type badge remains right-aligned |
| R-09 | Keyboard selection highlight (.vida-row-selected) still works correctly |
| R-10 | App icon names are passed from Go via the existing `vida_results_set_apps` signature extended to include icon names |

## Technical Notes

- `vida_results_set_apps` currently takes `const char **names, int n`. We need to add `const char **icons` parallel array so Go can pass icon names from the daemon's `AppResult.Icon` field.
- The daemon already returns `icon` in `AppResult` JSON; the Go UI code parses it but discards it (see `main.go` `case "app_list"`).
- In `ui.c`, use `gtk_image_new_from_gicon(g_themed_icon_new(icon_name), GTK_ICON_SIZE_LARGE)` then `gtk_image_set_pixel_size(image, 48)`.
- CSS: add `.vida-app-icon { border-radius: 6px; margin: 0 8px 0 12px; }` and `.vida-row { min-height: 56px; }`.
