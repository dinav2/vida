# SPEC-20260308-005: App Icons in Launcher Results

## 1. Overview

Render a 48×48 px app icon to the left of each app name in vida's launcher result rows. Icons are loaded from the app's `.desktop` Icon field via `GDesktopAppInfo`. A generic fallback icon is used when the app has none. This is a UI-only change: `ui.c` and the icon-passing in `cmd/vida-ui/main.go` are modified; the daemon is untouched.

---

## 2. Functional Requirements

### FR-01 · Icon Display
| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | Each app result row renders a `GtkImage` at 48×48 px, left-aligned |
| FR-01b | Icon is loaded from the app's `.desktop` Icon field via `g_app_info_get_icon()` on a `GDesktopAppInfo` |
| FR-01c | If icon name is empty, NULL, or fails to resolve, render `application-x-executable-symbolic` at 48px |
| FR-01d | Icon widget has CSS class `.vida-app-icon` |

### FR-02 · Layout
| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Row layout (left→right): icon — app name label (expands) — type badge "App" |
| FR-02b | Icon has 12px left margin, 8px right gap before label |
| FR-02c | Icon has 6px border-radius (rounded square) |
| FR-02d | Row min-height is 56px |

### FR-03 · Behaviour Preservation
| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | Keyboard arrow navigation selects/deselects rows correctly |
| FR-03b | Enter key launches the correct app regardless of icon presence |
| FR-03c | Clearing results removes all rows including their icon widgets |

### FR-04 · Data Flow
| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | `vida_results_set_apps` C function signature extended to `(GtkWidget *box, const char **names, const char **icons, int n)` |
| FR-04b | Go caller in `cmd/vida-ui/main.go` passes icon names extracted from `resp.Icons` (newline-separated) alongside names |
| FR-04c | Daemon already provides `icon` in each `AppResult`; Go UI must parse it from the IPC response |

## 3. Technical Requirements

### TR-01 · C Implementation (`cmd/vida-ui/ui.c`)

**CSS additions** (append to `VIDA_CSS`):
```c
".vida-app-icon {"
"  border-radius: 6px;"
"  margin: 0 8px 0 12px;"
"}"
/* Increase row height */
".vida-row {"
"  min-height: 56px;"  /* was 44px */
"}"
```

**`make_app_row` function** (replaces `make_row` for app results):
```c
static GtkWidget *make_app_row(const char *name, const char *icon_name) {
    GtkWidget *btn = gtk_button_new();
    gtk_widget_add_css_class(btn, "vida-row");
    gtk_button_set_has_frame(GTK_BUTTON(btn), FALSE);

    GtkWidget *hbox = gtk_box_new(GTK_ORIENTATION_HORIZONTAL, 0);
    gtk_widget_set_hexpand(hbox, TRUE);
    gtk_widget_set_valign(hbox, GTK_ALIGN_CENTER);

    /* Icon */
    GtkWidget *img;
    if (icon_name && *icon_name) {
        GIcon *gicon = g_themed_icon_new(icon_name);
        img = gtk_image_new_from_gicon(gicon);
        g_object_unref(gicon);
    } else {
        GIcon *gicon = g_themed_icon_new("application-x-executable-symbolic");
        img = gtk_image_new_from_gicon(gicon);
        g_object_unref(gicon);
    }
    gtk_image_set_pixel_size(GTK_IMAGE(img), 48);
    gtk_widget_add_css_class(img, "vida-app-icon");
    gtk_box_append(GTK_BOX(hbox), img);

    /* App name */
    GtkWidget *lbl = gtk_label_new(name);
    gtk_widget_add_css_class(lbl, "vida-row-label");
    gtk_label_set_ellipsize(GTK_LABEL(lbl), PANGO_ELLIPSIZE_END);
    gtk_label_set_xalign(GTK_LABEL(lbl), 0.0f);
    gtk_widget_set_hexpand(lbl, TRUE);
    gtk_box_append(GTK_BOX(hbox), lbl);

    /* Type badge */
    GtkWidget *type_lbl = gtk_label_new("App");
    gtk_widget_add_css_class(type_lbl, "vida-row-type");
    gtk_label_set_xalign(GTK_LABEL(type_lbl), 1.0f);
    gtk_box_append(GTK_BOX(hbox), type_lbl);

    gtk_button_set_child(GTK_BUTTON(btn), hbox);
    return btn;
}
```

**Updated `vida_results_set_apps` signature:**
```c
void vida_results_set_apps(GtkWidget *box, const char **names,
                            const char **icons, int n)
```

### TR-02 · Go Changes (`cmd/vida-ui/main.go`)

**IPC response parsing** — add `Icons` field to parsed result and pass it through:
```go
case "app_list":
    names := strings.Split(resp.Message, "\n")
    ids   := strings.Split(resp.IDs, "\n")
    execs := strings.Split(resp.Exec, "\n")
    icons := strings.Split(resp.Icons, "\n")  // NEW
    gtkIdle(func() {
        currentKind    = "app_list"
        currentAppIDs  = ids
        currentAppExecs = execs
        selectedIdx    = -1
        cnames := make([]*C.char, len(names))
        cicons := make([]*C.char, len(names))  // NEW
        for i, n := range names {
            cnames[i] = C.CString(n)
            defer C.free(unsafe.Pointer(cnames[i]))
            icon := ""
            if i < len(icons) { icon = icons[i] }
            cicons[i] = C.CString(icon)         // NEW
            defer C.free(unsafe.Pointer(cicons[i]))
        }
        C.vida_results_set_apps(gResults, &cnames[0], &cicons[0], C.int(len(cnames)))  // NEW
    })
```

### TR-03 · IPC Message (`internal/ipc/ipc.go`)

Add `Icons string` field to `Message` struct so the daemon's icon names round-trip through IPC:
```go
Icons string `json:"icons,omitempty"`
```

### TR-04 · Daemon (`cmd/vida-daemon/main.go`)

When building the `app_list` IPC response, populate `Icons` as a newline-joined string of icon names (parallel to `Message`/`IDs`/`Exec`).
