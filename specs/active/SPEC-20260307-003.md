# Spec: vida UI Styling — Mac-Style Launcher Aesthetic

**ID:** SPEC-20260307-003
**Stage:** SPEC
**Date:** 2026-03-07
**Priority:** P1
**PRD:** `specs/drafts/SPEC-20260307-003.md`

## Functional Requirements

### FR-01: Window Geometry
- **FR-01a:** Window is horizontally centered on the output
- **FR-01b:** Window is anchored left+right with a fixed max-width of 640px achieved via CSS `min-width`/`max-width` on the inner panel, or via margin
- **FR-01c:** Window sits at approximately 30% from the top of the screen (top margin ~200px or 30% of screen height)
- **FR-01d:** Window height is auto-sized to content; no fixed height set

### FR-02: Window Appearance
- **FR-02a:** Background is dark semi-transparent: `rgba(20, 20, 25, 0.92)`
- **FR-02b:** Rounded corners: 16px border-radius on the outer window/panel
- **FR-02c:** Subtle border: 1px solid `rgba(255, 255, 255, 0.08)`
- **FR-02d:** No GTK window decorations (already done; keep)
- **FR-02e:** The GTK window background is transparent so compositor shows the rounded corners against the desktop

### FR-03: Search Entry
- **FR-03a:** Font size ≥ 18px (target 20px)
- **FR-03b:** No visible entry border or box shadow — entry blends with panel
- **FR-03c:** Horizontal padding 20px, vertical padding 16px
- **FR-03d:** Placeholder text color: `rgba(255, 255, 255, 0.3)`
- **FR-03e:** Input text color: white `#ffffff`
- **FR-03f:** Entry background transparent (inherits panel background)
- **FR-03g:** A muted search icon appears to the left of the placeholder/text

### FR-04: Results Rows
- **FR-04a:** Each result row is at least 44px tall
- **FR-04b:** Result text: white, 15px
- **FR-04c:** Type label (e.g. "Calculator", "App", "Web"): right-aligned, 12px, `rgba(255,255,255,0.4)`
- **FR-04d:** Hover state: `rgba(255, 255, 255, 0.07)` background fill, 8px radius
- **FR-04e:** Separator line between entry and results: 1px `rgba(255,255,255,0.08)`
- **FR-04f:** Horizontal padding on rows: 20px

### FR-05: Typography
- **FR-05a:** Font family: `"Inter", system-ui, sans-serif`
- **FR-05b:** No bold weight used in results

## Technical Requirements

### TR-01: CSS via GTK4 CssProvider
- **TR-01a:** All styling is applied via a GTK4 `GtkCssProvider` loaded in `vida_build_window`
- **TR-01b:** CSS is embedded as a C string literal (no external file dependency)
- **TR-01c:** `gtk_widget_add_css_class` is used to target specific widgets

### TR-02: Transparent Window Background
- **TR-02a:** `gtk_widget_set_opacity` is NOT used (it affects all children)
- **TR-02b:** The GtkWindow gets CSS `background: transparent`
- **TR-02c:** The inner VBox panel gets the dark background and border-radius
- **TR-02d:** Requires compositor alpha channel support (standard on Hyprland/wlroots)

### TR-03: Window Positioning
- **TR-03a:** Layer-shell anchors: TOP only (not left+right) to allow horizontal centering
- **TR-03b:** `gtk_layer_set_margin` TOP: 200px (places panel ~30% from top on 1080p)
- **TR-03c:** The panel width is constrained via CSS `min-width: 640px; max-width: 640px`

### TR-04: Search Icon
- **TR-04a:** Implemented as a `GtkImage` with icon name `"system-search-symbolic"` prepended to the entry row
- **TR-04b:** Icon color: `rgba(255,255,255,0.3)` via CSS

### TR-05: Hover State
- **TR-05a:** Each result row is wrapped in a `GtkButton` with `flat` style class to get hover events, styled to remove button chrome

## Scenarios

### SCN-01: Window appears centered and styled
**Given** vida-ui is running and vida-daemon is active
**When** `vida show` is called
**Then** a dark rounded panel appears horizontally centered, ~30% from top
**And** the background shows desktop/wallpaper behind the rounded corners

### SCN-02: Search entry is prominent
**Given** the launcher is visible
**When** the user looks at the entry field
**Then** the text cursor is visible against the dark background
**And** the font is large and readable (≥18px)
**And** there is no visible entry box border

### SCN-03: Result type label appears
**Given** the user types `2+2`
**When** the calc result appears
**Then** the result row shows `4` on the left and `Calculator` muted on the right

### SCN-04: App results show type label
**Given** the user types `fire`
**When** app results appear
**Then** each row shows the app name on the left and `App` muted on the right

### SCN-05: Shortcut result shows type label
**Given** the user types `g hello`
**When** the shortcut result appears
**Then** the row shows the URL on the left and `Web` muted on the right
**And** an Open button is visible

## Architecture Notes

- All C changes in `cmd/vida-ui/ui.c`; Go changes minimal (result type labels passed from `main.go`)
- Result rows change from bare `GtkLabel` to `GtkButton > GtkBox > [GtkLabel + GtkLabel]`
- The existing `vida_results_set_label`, `vida_results_set_url`, `vida_results_set_apps` functions are rewritten
- `vida_build_window` gains CSS provider setup and restructured layout
- The outer `GtkWindow` becomes transparent; a new inner `GtkBox` (`panel`) holds the visual background
