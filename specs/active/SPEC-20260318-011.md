## 1. Overview

vida daemon continuously monitors the Wayland clipboard via `wl-paste --watch` and stores
entries in a new `clipboard_history` SQLite table. Typing `:cb` or `:clipboard` in the
palette closes the palette and opens a dedicated clipboard history window ‚Äî a separate
GTK4 layer-shell window with a search bar and scrollable entry list.

---

## 2. Functional Requirements

### FR-01 ¬∑ Clipboard Monitoring

| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | Daemon spawns `wl-paste --watch` on startup when `clipboard.enabled = true` |
| FR-01b | Each line of stdout from `wl-paste --watch` is treated as a new clipboard entry |
| FR-01c | If the new entry is identical to the most recent stored entry, it is discarded |
| FR-01d | Entries matching any `clipboard.ignore` regex are discarded silently |
| FR-01e | Image copies (binary / non-UTF-8 content) store `[image]` placeholder and are not pasted |
| FR-01f | If `wl-paste` is not installed, daemon logs a warning and skips clipboard monitoring |

### FR-02 ¬∑ Storage

| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | Entries are stored in a `clipboard_history` SQLite table in the existing vida database |
| FR-02b | Each entry stores: id, content (TEXT), copied_at (unix ts), pinned (0/1) |
| FR-02c | On each insert, unpinned entries beyond `max_entries` are pruned (oldest first) |
| FR-02d | On each insert, unpinned entries older than `max_age_days` are pruned |
| FR-02e | Pinned entries are exempt from both count and age pruning |

### FR-03 ¬∑ Palette Trigger

| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | `:cb` in the palette sends a `show_clipboard` IPC message and dismisses the palette |
| FR-03b | `:clipboard` behaves identically to `:cb` |
| FR-03c | `vida clipboard` CLI command sends `show_clipboard` to the daemon |
| FR-03d | Daemon broadcasts `show_clipboard` to all UI subscribers |

### FR-04 ¬∑ Clipboard Window

| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | The clipboard window is a second GTK4 layer-shell window inside `vida-ui` |
| FR-04b | Window is ~400px wide, ~500px tall, positioned same as the main palette |
| FR-04c | Window opens with search bar auto-focused |
| FR-04d | Entries are listed newest-first |
| FR-04e | Search bar filters entries live as you type (case-insensitive substring match) |
| FR-04f | Entries longer than 80 chars are truncated with `‚Ä¶` in the list display |
| FR-04g | Enter on selected entry writes full content to clipboard via `wl-copy`, closes window |
| FR-04h | `[image]` entries are displayed but Enter does nothing (no-op paste) |
| FR-04i | Escape closes the window |
| FR-04j | Window is hidden (not destroyed) after close; reused on next `show_clipboard` |

### FR-05 ¬∑ Entry Management

| Sub-ID | Requirement |
|--------|-------------|
| FR-05a | Delete key on a selected entry sends `clipboard_delete` IPC and removes it from the list |
| FR-05b | Ctrl+Shift+C on a selected entry sends `clipboard_pin` IPC (toggle); pinned entries show a `‚óè` prefix |
| FR-05c | Ctrl+Shift+X sends `clipboard_clear` IPC (clears all unpinned entries), list refreshes |

### FR-06 ¬∑ Configuration

| Sub-ID | Requirement |
|--------|-------------|
| FR-06a | `[clipboard] enabled = true` (default true) ‚Äî set to false to disable monitoring entirely |
| FR-06b | `[clipboard] max_entries = 500` ‚Äî maximum stored unpinned entries (default 500) |
| FR-06c | `[clipboard] max_age_days = 30` ‚Äî entries older than this are pruned (default 30, 0 = no limit) |
| FR-06d | `[clipboard] ignore = []` ‚Äî list of Go regex patterns; matching entries are not stored |

## 3. Technical Requirements

### TR-01 ¬∑ Package `internal/clipboard`

New package responsible for all clipboard history logic.

```go
// Entry is a single clipboard history item.
type Entry struct {
    ID       int64
    Content  string
    CopiedAt time.Time
    Pinned   bool
}

// Store manages clipboard history in SQLite.
type Store struct { /* wraps *db.DB */ }

func NewStore(database *db.DB) *Store

// Add stores a new entry, applying dedup, ignore patterns, and pruning.
// Returns (false, nil) if the entry was discarded.
func (s *Store) Add(content string, cfg ClipboardConfig) (bool, error)

// List returns entries matching query (empty = all), newest first, up to limit.
func (s *Store) List(query string, limit int) ([]Entry, error)

// Delete removes entry by ID.
func (s *Store) Delete(id int64) error

// TogglePin flips the pinned flag for entry with given ID.
func (s *Store) TogglePin(id int64) error

// Clear removes all unpinned entries.
func (s *Store) Clear() error

// ClipboardConfig is the relevant config subset.
type ClipboardConfig struct {
    Enabled    bool
    MaxEntries int
    MaxAgeDays int
    Ignore     []string // compiled to []*regexp.Regexp internally
}
```

### TR-02 ¬∑ Database Migration (`internal/db/db.go`)

Add `clipboard_history` table in `migrate()`:

```sql
CREATE TABLE IF NOT EXISTS clipboard_history (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    content   TEXT    NOT NULL,
    copied_at INTEGER NOT NULL,
    pinned    INTEGER NOT NULL DEFAULT 0
)
```

### TR-03 ¬∑ Config (`internal/config/config.go`)

Add `ClipboardConfig` struct and `Clipboard` field to `Config`:

```go
type ClipboardConfig struct {
    Enabled    bool     `toml:"enabled"`
    MaxEntries int      `toml:"max_entries"`
    MaxAgeDays int      `toml:"max_age_days"`
    Ignore     []string `toml:"ignore"`
}
```

Defaults applied in `Load()`: `Enabled=true`, `MaxEntries=500`, `MaxAgeDays=30`.

### TR-04 ¬∑ IPC Messages (`internal/ipc/ipc.go`)

New message types (no struct changes needed ‚Äî use existing fields):

| Type | Direction | Fields used | Meaning |
|------|-----------|-------------|---------|
| `show_clipboard` | daemon‚ÜíUI broadcast | ‚Äî | Open clipboard window |
| `hide_clipboard` | daemon‚ÜíUI broadcast | ‚Äî | Close clipboard window |
| `clipboard_list` | UI‚Üídaemon | `input` = search query | Request matching entries |
| `clipboard_entries` | daemon‚ÜíUI reply | `value` = newline-sep content, `ids` = newline-sep IDs, `icons` = newline-sep pinned flags ("1"/"0") | Entry list response |
| `clipboard_paste` | UI‚Üídaemon | `id` = entry ID | Write entry to clipboard via wl-copy |
| `clipboard_delete` | UI‚Üídaemon | `id` = entry ID | Delete entry |
| `clipboard_pin` | UI‚Üídaemon | `id` = entry ID | Toggle pin on entry |
| `clipboard_clear` | UI‚Üídaemon | ‚Äî | Clear all unpinned entries |

### TR-05 ¬∑ Daemon (`cmd/vida-daemon/main.go`)

```go
// New fields on daemon struct:
clipStore    *clipboard.Store
clipWatcher *exec.Cmd  // wl-paste --watch subprocess

// New methods:
func (d *daemon) startClipboardWatcher()  // spawned in run() if clipboard.enabled
func (d *daemon) handleClipboardMessage(msg ipc.Message, reply ipc.ReplyFunc)
```

`startClipboardWatcher` runs `wl-paste --watch --no-newline` and reads stdout line by line,
calling `d.clipStore.Add(line, cfg)` for each.

`handleMessage` switch gains cases: `clipboard_list`, `clipboard_paste`, `clipboard_delete`,
`clipboard_pin`, `clipboard_clear`.

`show_clipboard` / `hide_clipboard` are sent via `srv.Broadcast()` (existing mechanism,
same as `show`/`hide`).

Router command handling: when `KindCommandList` result has `CommandQuery` of `cb` or
`clipboard`, daemon sends `show_clipboard` broadcast and `{"type":"ok"}` reply to palette.

### TR-06 ¬∑ UI (`cmd/vida-ui/main.go` + C layer)

- New `clipboardWindow` GTK4 widget (GTK4 `GtkWindow` or second layer-shell surface)
- Subscribes to IPC; on `show_clipboard` message: fetch `clipboard_list` and show window
- On `hide_clipboard` or Escape: hide window
- On Enter: send `clipboard_paste` with selected entry ID
- On Delete key: send `clipboard_delete` with selected entry ID
- On Ctrl+Shift+C: send `clipboard_pin` with selected entry ID, refresh list
- On Ctrl+Shift+X: send `clipboard_clear`, refresh list
- Search bar change: send `clipboard_list` with current query, refresh list

### TR-07 ¬∑ CLI (`cmd/vida/main.go`)

New subcommand `clipboard`:
```
vida clipboard   ‚Üí sends {"type":"show_clipboard"} via IPC
```

## 4. Test Scenarios (Given / When / Then)

### SCN-01 ¬∑ `:cb` opens clipboard window
**Given** the palette is open
**When** the user types `:cb` and presses Enter
**Then** the palette closes and `show_clipboard` is broadcast to UI

### SCN-02 ¬∑ `:clipboard` is equivalent to `:cb`
**Given** the palette is open
**When** the user types `:clipboard` and presses Enter
**Then** same behaviour as SCN-01

### SCN-03 ¬∑ Clipboard window shows entries newest-first
**Given** the clipboard window is open with entries A (older) and B (newer)
**When** the window renders
**Then** B appears above A

### SCN-04 ¬∑ Live search filters entries
**Given** the clipboard window is open with entries "hello world" and "foo bar"
**When** the user types "foo" in the search bar
**Then** only "foo bar" is shown

### SCN-05 ¬∑ Enter pastes and closes
**Given** the clipboard window is open with "hello world" selected
**When** the user presses Enter
**Then** "hello world" is written to clipboard via `wl-copy` and the window closes

### SCN-06 ¬∑ Escape closes the window
**Given** the clipboard window is open
**When** the user presses Escape
**Then** the window hides

### SCN-07 ¬∑ `vida clipboard` CLI opens window
**Given** the daemon is running
**When** `vida clipboard` is run
**Then** daemon broadcasts `show_clipboard`

### SCN-08 ¬∑ Daemon stores new clipboard entry
**Given** the clipboard watcher is running
**When** the user copies "new text"
**Then** `clipboard_history` contains an entry with content "new text"

### SCN-09 ¬∑ Consecutive duplicate not stored
**Given** the most recent entry is "foo"
**When** the user copies "foo" again
**Then** no new row is inserted

### SCN-10 ¬∑ max_entries pruning
**Given** `max_entries = 3` and 3 unpinned entries exist
**When** a fourth entry is copied
**Then** the oldest unpinned entry is deleted; 3 rows remain

### SCN-11 ¬∑ max_age_days pruning
**Given** `max_age_days = 1` and an unpinned entry is 2 days old
**When** a new entry is copied
**Then** the 2-day-old entry is deleted

### SCN-12 ¬∑ Pinned entries survive pruning
**Given** `max_entries = 1`, 1 unpinned entry, and 1 pinned entry
**When** a new entry is copied
**Then** the pinned entry remains; the unpinned entry is deleted

### SCN-13 ¬∑ Delete removes single entry
**Given** an entry is selected in the clipboard window
**When** the user presses Delete
**Then** that entry is removed from the list and from the database

### SCN-14 ¬∑ Clear removes all unpinned entries
**Given** 3 unpinned and 1 pinned entry in clipboard history
**When** the user presses Ctrl+Shift+X
**Then** 0 unpinned entries remain; the pinned entry remains

### SCN-15 ¬∑ Ignore pattern suppresses entry
**Given** `ignore = ["^sk-"]`
**When** text matching `sk-abc123` is copied
**Then** it is not stored

### SCN-16 ¬∑ Long entries truncated in list
**Given** an entry of 200 characters
**When** it is shown in the clipboard window
**Then** the displayed text is truncated to 80 chars with `‚Ä¶`; full text is pasted on Enter

### SCN-17 ¬∑ Image entries show placeholder
**Given** the user copies an image
**When** the entry is stored and displayed
**Then** `[image]` is shown; Enter on this entry is a no-op

## 5. Acceptance Criteria

| ID | Criteria |
|----|----------|
| AC-01 | `internal/clipboard` package compiles with zero CGo dependencies |
| AC-02 | All 17 scenarios covered by table-driven unit or integration tests |
| AC-03 | `wl-paste` absence causes a logged warning, not a crash |
| AC-04 | Clipboard window is hidden, not destroyed, between uses (no GTK recreation overhead) |
| AC-05 | `vida clipboard` sends `show_clipboard` and exits 0 when daemon is running |
| AC-06 | Ignore patterns are compiled once at config load; invalid regex causes logged warning, not crash |
| AC-07 | All clipboard IPC handlers return an `ok` or `error` reply (no silent hangs) |

## 6. Out of Scope

- Syncing clipboard history across machines
- Image thumbnails (placeholder text only for now)
- Per-entry timestamps shown in UI
- Wayland primary selection (middle-click clipboard)
