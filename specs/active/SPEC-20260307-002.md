# SPEC-20260307-002 — vida Search Input Wiring

**Status:** SPEC
**Priority:** P1
**Created:** 2026-03-07
**PRD:** `specs/drafts/SPEC-20260307-002.md`

## 1. Functional Requirements

### FR-01 · Query on keystroke
| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | Every change to the entry widget triggers a `query` IPC message with the current text |
| FR-01b | Calc and shortcut queries are sent immediately (no debounce) |
| FR-01c | App-search queries are sent immediately |
| FR-01d | AI queries are sent after 80 ms idle (debounce to avoid flooding) |
| FR-01e | Empty input sends no query; results area is cleared |

### FR-02 · Result rendering
| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | `kind: calc` → display value string in a single-line result label |
| FR-02b | `kind: shortcut` → display expanded URL with an "Open" button |
| FR-02c | `kind: app_list` → display a vertical list of app name rows (icon + name) |
| FR-02d | `kind: ai_stream` → display a text area that grows as tokens arrive |
| FR-02e | `kind: empty` or `kind: cancelled` → clear the results area |
| FR-02f | Each new query replaces the entire results widget (no partial updates to old result) |

### FR-03 · App launch
| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | Enter key (or click) on an app row launches the app via `gio open` subprocess |
| FR-03b | After launch, the window hides |
| FR-03c | `apps.ExpandExec` is used to strip `%u/%f/%U/%F` placeholders before launch |

### FR-04 · Enter key behaviour
| Sub-ID | Requirement |
|--------|-------------|
| FR-04a | Calc result: Enter copies the value to clipboard |
| FR-04b | Shortcut result: Enter opens the URL in the default browser |
| FR-04c | App list: Enter launches the top result |
| FR-04d | AI result: Enter copies accumulated text to clipboard |

### FR-05 · Window show resets state
| Sub-ID | Requirement |
|--------|-------------|
| FR-05a | On receiving a `show` broadcast, clear the entry widget |
| FR-05b | On receiving a `show` broadcast, clear the results area |
| FR-05c | On show, move focus to the entry widget |

### FR-06 · AI cancellation
| Sub-ID | Requirement |
|--------|-------------|
| FR-06a | When the window hides (Escape or `hide` broadcast), any in-flight AI query is cancelled |
| FR-06b | Cancellation is sent as a `cancel` IPC message with `id` matching the query |
| FR-06c | The daemon aborts the in-flight HTTP request when it receives `cancel` |
| FR-06d | A new query supersedes any previous in-flight query (implicit cancel of old) |

### FR-07 · Debounce timing
| Sub-ID | Requirement |
|--------|-------------|
| FR-07a | Calc/shortcut/app results visible within 50 ms of keystroke |
| FR-07b | AI query sent no earlier than 80 ms after last keystroke |

## 2. Technical Requirements

### TR-01 · Thread safety
| Sub-ID | Requirement |
|--------|-------------|
| TR-01a | All GTK widget reads/writes happen on the GLib main thread |
| TR-01b | IPC responses received in goroutines are dispatched to GTK via `g_idle_add` |

### TR-02 · IPC protocol extensions
| Sub-ID | Requirement |
|--------|-------------|
| TR-02a | `query` message: `{"type":"query","id":"<uuid>","input":"<text>"}` |
| TR-02b | Daemon `result` response: `{"type":"result","id":"<uuid>","kind":"<kind>","value":"...","url":"..."}` |
| TR-02c | AI streaming: daemon sends multiple `{"type":"token","id":"<uuid>","value":"<chunk>"}` messages followed by `{"type":"done","id":"<uuid>"}` |
| TR-02d | `cancel` message: `{"type":"cancel","id":"<uuid>"}` — daemon stops streaming for that ID |

### TR-03 · AI streaming connection
| Sub-ID | Requirement |
|--------|-------------|
| TR-03a | UI uses `ipc.PersistentConn` for AI queries to receive token stream |
| TR-03b | Non-AI queries use `ipc.Conn` (one-shot) |

### TR-04 · Result widget lifecycle
| Sub-ID | Requirement |
|--------|-------------|
| TR-04a | Results container is a `GtkBox` child of the main window |
| TR-04b | On each new result, remove old child widget and add new one |

### TR-05 · Daemon cancel support
| Sub-ID | Requirement |
|--------|-------------|
| TR-05a | Daemon tracks in-flight AI queries by ID in a `map[string]context.CancelFunc` |
| TR-05b | On `cancel` message, call the cancel function and remove the entry |
| TR-05c | On `done` send, remove the entry |

### TR-06 · App launch
| Sub-ID | Requirement |
|--------|-------------|
| TR-06a | Launch command: `exec.Command("sh", "-c", expandedExec)` |
| TR-06b | Process is started detached (no wait) |

## 3. Scenarios

### SCN-01 · Calculator result appears
```
Given: vida-ui is visible, entrada field focused
When:  User types "sqrt(144)"
Then:  Result label shows "12" within 50 ms
And:   No IPC query with kind=ai_stream is sent
```

### SCN-02 · Shortcut result appears
```
Given: vida-ui is visible
And:   config has shortcut g = "https://www.google.com/search?q=%s"
When:  User types "g linux kernel"
Then:  Result shows the expanded URL
And:   An "Open" button is visible
```

### SCN-03 · App search result appears
```
Given: vida-ui is visible
And:   Firefox is installed (/usr/share/applications/firefox.desktop)
When:  User types "fire"
Then:  App list shows at least one entry containing "Firefox"
```

### SCN-04 · App launch
```
Given: App list shows Firefox
When:  User presses Enter
Then:  Firefox process starts
And:   vida-ui window hides
```

### SCN-05 · Enter copies calc result
```
Given: Result label shows "12"
When:  User presses Enter
Then:  "12" is in the clipboard
```

### SCN-06 · Empty input clears results
```
Given: Results area shows a calc result
When:  User clears the entry
Then:  Results area is empty
And:   No query is sent
```

### SCN-07 · AI query streams tokens
```
Given: ANTHROPIC_API_KEY or OPENAI_API_KEY is set
When:  User types "explain what an inode is" and waits 80 ms
Then:  Tokens appear progressively in the result area
And:   UI remains responsive during streaming
```

### SCN-08 · Cancel on hide
```
Given: An AI query is streaming
When:  User presses Escape
Then:  Window hides immediately
And:   Daemon receives cancel message for the query ID
And:   In-flight HTTP request is aborted
```

### SCN-09 · New query cancels previous AI
```
Given: An AI query for "foo" is streaming
When:  User clears input and types "bar"
Then:  Daemon receives cancel for the "foo" query ID
And:   A new query for "bar" is started
```

### SCN-10 · Window show resets state
```
Given: vida-ui is hidden with a previous result visible
When:  vida show is executed
Then:  Entry field is empty
And:   Results area is empty
And:   Entry field has keyboard focus
```

## 5. Architecture Notes

### Changes to existing packages

**`cmd/vida-daemon/main.go`**
- Add `cancel` message handler (FR-06c, TR-05)
- Track in-flight AI contexts by query ID
- Add token-streaming response for AI queries (TR-02c)

**`cmd/vida-ui/main.go` + `ui.c`**
- Add results container (`GtkBox`) below entry
- Wire `GtkEntry` `changed` signal → debounced query goroutine
- Add result renderers: calc label, shortcut box, app list box, AI text view
- Wire `key-pressed` Enter → act on current result
- Wire show broadcast → reset entry + results

### New flow

```
keystroke
  → GtkEntry changed (GLib thread)
  → Go goroutine: debounce timer
  → ipc.Conn.Send(query)          ← one-shot for calc/shortcut/app
  → OR ipc.PersistentConn         ← for AI streaming
  → result/token received
  → g_idle_add → GTK widget update
```

### File map

| File | Changes |
|------|---------|
| `cmd/vida-ui/main.go` | Query goroutine, debounce, result dispatch, cancel logic |
| `cmd/vida-ui/ui.c` | Result widget builders, g_idle_add wrappers, app launch |
| `cmd/vida-daemon/main.go` | cancel handler, AI streaming with token sends, context tracking |
| `internal/ipc/ipc.go` | No changes needed |
| `internal/router/router.go` | No changes needed |
