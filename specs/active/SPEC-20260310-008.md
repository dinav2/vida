# SPEC-20260310-008: Unit Conversion

## 1. Overview

vida detects unit conversion queries typed directly in the palette and displays the result inline — no command prefix, no AI call, no network. The user types `5 cm to in` and immediately sees `5 cm → 1.97 in`. Currency is out of scope (SPEC-20260310-009).

---

## 2. Functional Requirements

### FR-01 · Detection & Routing

| Sub-ID | Requirement |
|--------|-------------|
| FR-01a | Conversion detection runs after command mode and before calc |
| FR-01b | A query matches if it has the form `<number> <unit> to <unit>` or `<number> <unit> in <unit>` (case-insensitive) |
| FR-01c | `→` is accepted as an alias for `to` |
| FR-01d | Whitespace between number and unit is optional: `5cm to in` and `5 cm to in` both match |
| FR-01e | If both units are recognised but the query doesn't match (e.g. mixing families), no result; routing continues to calc |
| FR-01f | If either unit is unrecognised, no conversion result; routing falls through |

### FR-02 · Unit Families

| Sub-ID | Requirement |
|--------|-------------|
| FR-02a | **Length**: mm, cm, m, km, in/inch/inches, ft/foot/feet, yd/yard, mi/mile/miles |
| FR-02b | **Weight**: mg, g, kg, t/tonne, oz, lb/lbs/pound/pounds |
| FR-02c | **Temperature**: C/°C/celsius, F/°F/fahrenheit, K/kelvin |
| FR-02d | **Area**: mm2/mm², cm2/cm², m2/m², km2/km², in2/in², ft2/ft², acre, ha/hectare |
| FR-02e | **Volume**: ml, l/litre/liter, floz/fl-oz, cup/cups, pt/pint, qt/quart, gal/gallon |
| FR-02f | **Speed**: km/h/kph, mph, m/s, knot/knots |
| FR-02g | **Data**: B, KB, MB, GB, TB, PB — base-2 (1 KB = 1024 B) |
| FR-02h | **Time**: s/sec/second/seconds, min/minute/minutes, h/hr/hour/hours, d/day/days, wk/week/weeks |

### FR-03 · Result Display

| Sub-ID | Requirement |
|--------|-------------|
| FR-03a | Result row text: `<input> <from> → <result> <to>` |
| FR-03b | Result row shows a `Convert` badge (not `Calculator`) |
| FR-03c | Numeric result trims trailing zeros: `1.970` → `1.97`, `2.0` → `2` |
| FR-03d | Temperature displays degree symbol: `32°C → 89.6°F` |
| FR-03e | Enter or Ctrl+C copies the result value to clipboard |
| FR-03f | Incompatible unit families (e.g. `5 km to kg`) produce no result |

## 3. Technical Requirements

### TR-01 · Package `internal/convert`

```go
// IsConversion reports whether s looks like a unit conversion query.
func IsConversion(s string) bool

// Convert evaluates a unit conversion query.
// Returns the formatted display string and true on success.
// Returns "", false if units are unknown, incompatible, or input is malformed.
func Convert(s string) (string, bool)
```

Pattern (conceptual):
```
^(\d+(?:\.\d+)?)\s*([^\s]+)\s+(?:to|in|→)\s+(.+)$
```

### TR-02 · Router Change (`internal/router/router.go`)

New `KindConvert` kind. Priority between command mode (0) and calc (1):

```go
KindConvert Kind = "convert"

// In Route():
if convert.IsConversion(input) {
    if result, ok := convert.Convert(input); ok {
        return Result{Kind: KindConvert, CalcValue: result}
    }
}
```

### TR-03 · Daemon (`cmd/vida-daemon/main.go`)

`KindConvert` handled identically to `KindCalc` — send `result` message with `kind: "convert"`.

### TR-04 · UI (`cmd/vida-ui/ui.c` + `main.go`)

`convert` result kind renders with a `Convert` badge. Copy-on-Enter behaviour identical to calc.
